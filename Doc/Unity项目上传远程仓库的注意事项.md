# Unity项目上传远程仓库的注意事项

1. Unity项目中的所有文件并**不需要完全上传**到远程仓库，或者说全部上传仓库甚至会造成**浪费空间资源**的情况。
   1. 一般unity文件目录构成：
   
   ![上传项目注意事项 1-1](Image\上传项目注意事项 1-1.png)
   
     其中红色框体内的文件是必须上传的
   
     - Assets：Unity项目中所有资源的存放文件
   
     - BuildSettings:与打包设置相关，应该包含你设置的要打包的场景设置等
   
     - Packages：标注你项目中需要使用的各种unity API
   
     - ProjectSetting：Layer、Tag等以及其他内容的全局设置保存其中
   
     - UserSetting：Unity界面布局等设置保存其中
   
       
   
     绿色边框内的文件是git相关文件，也需上传：
   
     - **.git**: 仓库文件本体，**删除该文件相当于删除仓库**，其中保存所有commit、分支的文件修改信息。
     - .gitignore:其中可配置无需git文件管理的内容，我们也是通过配置.gitignore来防止上传不必要的文件。
   
2. .gitignore配置：

   本质上.gitignore就是一个文本文件，直接打开进行编辑即可

   ![上传目录注意事项 1-2](Image\上传目录注意事项 1-2.png)

   1. 忽略某个文件：

      直接添加目标文件的相对地址到.gitignore中即可，

      例如：

      - `.vs`  -->直接忽略.vs文件
      - `./vs` -->与上一条作用一致，但是使用了相对地址写法 （`./`代表当前目录， `../`代表上一级目录）
      - `*.XXX`，忽略所有以`./XXX`结尾的所有文件。(这种是正则表达式的写法)

   2. 忽略某目录及其下所有文件

      书写该目录相对地址并在其后添加 "/"即可

      例如：

      - `./Library/ ` --> 直接忽略Library目录与其下所有文件

   3. 其他

      当然还有其他的写法，比如忽略某目录下所有文件，但是也可上传其中某些类型的文件

      还有大小写兼容，以及类似正则表达式的写法。感兴趣或者有需求的可以上网查阅
      
      

3. 我也上传了这个.gitignore文件，一般来说，你们开新仓库之后最好加上这个文件。

4. 特别注意：上传类似github、gitee等远程仓库，在**没有出钱**的时候，上传文件是存在各种限制的。例如：

   1. github应该要求总大小不能超过1G还是2G来着，且上传的单个文件不能超过100M（如果有可以使用lfs解决）。
   2. gitee限制就更多了，在前面的基础上不能使用lfs服务（在你不出钱的情况下，哈哈哈哈哈）
   3. gitcode还行，没啥限制，但是听说保密性不好，但是总归能用。你们到时候发现实在没招了可以来试试。
